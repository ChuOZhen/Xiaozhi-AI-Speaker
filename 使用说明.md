# ESP32 AI 语音助手 — 使用说明

## 项目简介

基于 ESP32-S3 的智能语音助手，支持"你好小智"唤醒、语音识别、大模型对话、语音播报，具备联网搜索和连续多轮对话能力。

## 工作流程

```
唤醒"你好小智" → 录音 → ASR语音识别 → LLM对话(支持联网搜索) → TTS语音合成 → ESP32播放 → 继续监听
```

## 硬件要求

| 组件 | 型号 |
|------|------|
| 主控 | ESP32-S3 |
| 麦克风 | INMP441 (I2S_0) |
| 功放+喇叭 | MAX98357A (I2S_1) |
| 唤醒词 | "你好小智" |

## 环境准备

### Python 服务端

```bash
cd esp32-ai-voice
source .venv/bin/activate
pip install -r requirements.txt   # 如果没有 requirements.txt
# 手动安装依赖:
pip install websockets edge-tts numpy aiohttp httpx
```

### 环境变量（~/.bashrc 中配置）

```bash
# LLM (DeepSeek)
export OPENAI_API_KEY="sk-xxxxxxxx"
export OPENAI_BASE_URL="https://api.deepseek.com/v1"

# 百度 ASR
export BAIDU_VOICE_APP_ID="xxxxxxxx"
export BAIDU_VOICE_API_KEY="xxxxxxxx"
export BAIDU_VOICE_SECRET_KEY="xxxxxxxx"

# 博查联网搜索（可选，不配置则不启用搜索）
export BOCHA_API_KEY="sk-xxxxxxxx"
```

### ESP32 固件

```bash
cd esp32-ai-voice
source ~/esp/esp-idf/export.sh
idf.py build flash monitor
```

### 网络拓扑

ESP32 通过 WiFi 连接局域网，与运行 `proxy_server.py` 的 PC 在同一子网。

> **推荐使用桥接模式**：VMware 网络适配器改为桥接，VM 直接获得局域网 IP，ESP32 可直接连接。

ESP32 固件中 `WS_URI` 需指向服务端 IP：
```c
// main/main.cc
#define WS_URI "ws://<服务端IP>:8765/ws/esp32"
```

## 启动

### 1. 启动服务端

```bash
cd esp32-ai-voice
source .venv/bin/activate
python3 proxy_server.py
```

启动后看到：
```
监听: ws://0.0.0.0:8765/ws/esp32
ASR: baidu
LLM: openai
TTS: baidu
等待 ESP32 连接...
```

### 2. ESP32 上电

ESP32 自动连接 WiFi → WebSocket → 打印：
```
✅ 系统就绪！说 '你好小智' 唤醒
```

### 3. 开始对话

对着麦克风说"你好小智"唤醒，然后说出你想问的问题。

## 对话示例

```
用户: 你好小智 → "你好！"（欢迎音）
用户: 今天天气怎么样
ASR: 今天天气怎么样
LLM: 搜索天气 → "今天北京晴，2026度，适合..."
TTS: 语音播报

用户: 我叫李华
ASR: 我叫李华
LLM: "你好李华！记住了"
TTS: 语音播报

用户: 我是谁            ← 多轮上下文
ASR: 我是谁
LLM: "你是李华啊！刚才你告诉我的"
TTS: 语音播报

（不说话10秒，自动播放"拜拜"进入睡眠，需重新唤醒）
```

## 问答模式（Quiz）

说出触发短语进入问答模式。系统会先问你想考什么主题，然后用 LLM 动态生成 3 道题，逐题问答，语义判题，3 题结束后自动退出。

### 触发短语

| 短语 |
|------|
| "问我一个问题" |
| "考考我" |
| "能不能考考我" |
| "来个问题" / "来一个问题" |
| "出个题" / "出一道题" |
| "测试我" |
| "给我一道题" |

### 交互示例

```
用户: 你好小智                  → 唤醒
ESP32: "你好！"

用户: 考考我                    → 触发 Quiz
ESP32: "你想让我考你哪一方面的知识？比如 ESP32、Python、英语、物理？"

用户: ESP32                    → 指定主题
ESP32: "好的，关于ESP32，第一题：ESP32-S3是什么架构的芯片？"

用户: Xtensa架构               → 语义判题正确
ESP32: "回答正确！下一题：ESP32-S3主频最高多少？"

用户: 200MHz                   → 语义判题错误
ESP32: "不太准确，正确答案是240MHz。下一题：..."

ESP32: "三道题已完成，问答结束！"  → 自动退出，回到普通聊天
```

### 状态机

```
ASK_TOPIC → 问用户要什么主题
    ↓
GENERATE → LLM 动态生成 3 道题（JSON）
    ↓
ASKING  → 播报当前题
    ↓
JUDGE   → LLM 语义判题（理解同义表达）
    ↓
NEXT    → 下一题 or FINISHED
```

### 工作流程

```
ASR 识别文本
    ↓
Intent Router（意图分类）
    ├── QUIZ  → Quiz Agent（独立状态机）
    │               ↓
    │          询问主题 → LLM出题 → 逐题问答 → LLM语义判题 → TTS播报
    │
    └── NORMAL → 正常 LLM 对话（不受影响）
```

> **Quiz 完全独立**：使用独立 LLM 实例（不影响聊天上下文），不修改 ESP32 状态机，不侵入主 Pipeline。

## 配置说明

### 切换 ASR 提供商

```bash
# 百度（默认，中文识别稳定）
python3 proxy_server.py

# OpenAI Whisper（需设置 WHISPER_BASE_URL）
WHISPER_BASE_URL=https://api.openai.com/v1 ASR_PROVIDER=whisper python3 proxy_server.py
```

`local_server/config.py` 中 `ASR_PROVIDER` 可选值：`baidu` / `whisper` / `volcengine`

### 切换 LLM 提供商

修改 `OPENAI_BASE_URL` 环境变量即可，任何 OpenAI 兼容 API 均可：
- DeepSeek: `https://api.deepseek.com/v1`
- 豆包（火山引擎ARK）: `https://ark.cn-beijing.volces.com/api/v3`
- 通义千问: DashScope 兼容地址

### 切换 TTS 提供商

`local_server/config.py` 中 `TTS_PROVIDER` 可选值：`baidu`（推荐，国内稳定）/ `edge_tts`（免费，需访问微软）/ `volcengine`

```bash
# 百度 TTS（推荐，与ASR共用Key）
TTS_PROVIDER=baidu python3 proxy_server.py

# Edge-TTS（免费，但国内可能被墙）
python3 proxy_server.py
```

### 联网搜索

博查 API Key 配置后自动启用。触发搜索的关键词：
- 时间类：今天、现在、几号、星期
- 实时类：新闻、最新、最近、天气
- 事实类：谁是、现任、价格、汇率

不触发搜索的日常对话直接 LLM 回复，节省 API 调用。


### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| ESP32连不上服务端 | IP不对或网络不通 | 检查 WS_URI，ping 测试 |
| ASR识别中文变英文 | dev_pid 不对 | config.py 确认 dev_pid=1537 |
| TTS没声音 | edge-tts访问Bing被墙 | 自动重试3次，或换TTS |
| LLM回复日期错误 | 系统提示词没有注入日期 | 当前已修复，动态注入 |
| 多轮对话丢失上下文 | 历史超长或被清理 | 默认保留20轮，断线会清理 |
| 联网搜索不触发 | Key未配置或关键词不匹配 | 检查 BOCHA_API_KEY |
| Quiz没反应 | 没说出触发短语 | 说"问我一个问题"等 |
| Quiz TTS没声音 | edge-tts被墙 | 切到百度TTS：`TTS_PROVIDER=baidu` |

## 核心架构

```
ESP32-S3 固件 (C++)
├── WebSocket Client → ws://PC_IP:8765/ws/esp32
├── WakeNet: "你好小智"唤醒词检测
├── AFE: 音频前端(VAD静音检测+AGC自动增益)
├── 状态机: SLEEP→LISTENING→RECORDING→WAITING→PLAYING→CONTINUOUS_LISTEN
├── I2S_0 (INMP441): 16kHz 16bit 单声道录音
└── I2S_1 (MAX98357A): 音频播放

Python 服务端 (proxy_server.py)
├── WebSocket Server: 0.0.0.0:8765
├── AudioSession: 录音缓冲 + watchdog超时(3秒)
├── Intent Router: 意图分类 → NORMAL / QUIZ
├── Pipeline: ASR → 路由 → (LLM → TTS) 或 (Quiz Agent)
├── Quiz Agent: 动态问答状态机(LLM出题+语义判题)
├── Search: 博查联网搜索(自动判断是否触发)
└── 多轮上下文: conversation_history[client_id](最多20轮)
```
